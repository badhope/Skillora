## Code of Conduct

By participating in this project, you agree to abide by our code of conduct:

- Be respectful and inclusive
- Be welcoming to newcomers
- Focus on what's best for the community
- Show empathy toward other contributors

## How Can I Contribute?

### 🐛 Report Issues

If you find bugs, inconsistencies, or have suggestions for improvements:

1. Check if the issue already exists
2. Open a new issue with clear description
3. Include relevant context and examples

### ✨ Add New Prompts

Help grow our prompt collection:

- Platform-specific prompts
- Use-case templates
- Best practices guides
- Multilingual prompts

### 📚 Improve Documentation

Help make our docs better:

- Fix typos or clarifications
- Add missing examples
- Improve explanations
- Add tutorials

### 💡 Suggest Enhancements

Have ideas for how to make PromptHub better? Let us know!

## Getting Started

### 1. Fork & Clone

```bash
# Fork the repository
# Clone your fork
git clone https://github.com/[your-username]/PromptHub.git
cd PromptHub
```

### 2. Create Branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Changes

Add or modify files following our style guide

### 4. Commit & Push

```bash
git add .
git commit -m "feat: add your feature description"
git push origin feature/your-feature-name
```

### 5. Open Pull Request

Submit a PR from your branch to our `main` branch

## Contribution Guidelines

### Adding a New Prompt

1. **Choose the Right Location**

   - `international/[platform]/` for international platforms
   - `chinese/[platform]/` for Chinese platforms
   - `opensource/` for open-source curated content
   - `examples/` for templates and examples

2. **Follow Our Format**

   ```markdown
   ## Prompt Title
   
   Brief description of when to use this prompt.
   
   ```
   Your full prompt here
   ```
   
   ### Usage Tips
   - Tip 1
   - Tip 2
   
   ### Examples
   Example usage if applicable.
   ```

3. **Include Metadata**

   - Supported platforms
   - Use cases
   - Tags
   - Author/credit if applicable

### Improving Existing Content

1. **Fix Issues**
   - Correct typos or grammar
   - Improve clarity
   - Update outdated information

2. **Enhance Content**
   - Add better examples
   - Expand explanations
   - Include cross-platform notes

3. **Refactor**
   - Better organization
   - Clearer structure
   - Improved readability

## Submitting Changes

### Pull Request Process

1. Update your fork to the latest `main`
2. Create a clear PR title: `feat: add [description]` or `fix: [description]`
3. Fill out the PR template completely
4. Wait for review and address feedback
5. Get approved and merged!

### Commit Messages

Follow conventional commits format:

```
feat: add new llama prompts
fix: correct typo in documentation
docs: update contributing guide
chore: update configuration
```

## Style Guide

### Markdown Style

- Use `#` for main headers
- Use `##` for section headers
- Use `-` for bullet points
- Use code blocks with language tags
- Keep lines under 80 characters

### File Organization

- Clear, descriptive filenames
- All lowercase with hyphens
- Proper directory structure

### Prompt Quality

- Clear instructions
- Well-structured
- Tested if possible
- Cross-platform notes
